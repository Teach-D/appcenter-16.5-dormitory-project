이력서 항목을 분석하고, 이 프로젝트에 적용 가능한 기능으로 발전시켜 이슈 생성까지 연결해줘.

이력서 항목: $ARGUMENTS

## 진행 순서

### 1단계: 이력서 항목 분석 및 발전

입력받은 이력서 항목을 아래 관점에서 분석한다:

**분석 항목**
- **핵심 기술/경험**: 이 항목이 어필하는 기술 역량은 무엇인가?
- **약점**: 현재 문장에서 모호하거나 임팩트가 부족한 부분은?
- **수치화 가능성**: 성능, 속도, 처리량 등 수치로 표현할 수 있는 요소가 있는가?
- **기술 키워드**: 면접관이 주목할 기술 키워드가 포함되어 있는가?

그 다음, 개선된 이력서 문장을 **3가지 버전**으로 제안한다.

**출력 형식 규칙 (모든 버전 공통)**:
각 버전은 반드시 아래 3줄 구조를 지킨다. 한 줄은 한 문장, 줄바꿈 없이 작성.

```
[버전 A - 기술 중심]
[문제] {어떤 기술적 상황/한계가 있었는지 한 줄}
[해결] {어떤 기술로 어떻게 해결했는지 한 줄 — 기술 스택과 구현 방식 강조}
[성과] {수치 포함 결과 한 줄}

[버전 B - 성과 중심]
[문제] {어떤 기술적 상황/한계가 있었는지 한 줄}
[해결] {어떤 기술로 어떻게 해결했는지 한 줄}
[성과] {수치와 비즈니스 임팩트를 강조한 한 줄}

[버전 C - 문제 해결 중심]
[문제] {근본 원인까지 파고든 문제 정의 한 줄}
[해결] {문제 원인에 직접 대응한 해결 방식 한 줄}
[성과] {수치 포함 결과 한 줄}
```

---

### 2단계: 프로젝트 적용 가능성 검토

이력서 항목의 기술/경험이 UniDorm 프로젝트에 **실제로 구현 가능한지** 검토한다.

**검토 기준**
- 기존 도메인/기능과의 연관성
- 현재 기술 스택으로 구현 가능한지 (Spring Boot, MySQL, Redis, FCM, WebSocket)
- 이미 구현된 기능과 중복되지 않는지
- 기숙사 서비스 맥락에서 실제로 유용한 기능인지

**결과 출력 형식**
```
## 적용 가능성 검토

**적용 가능 여부**: ✅ 가능 / ⚠️ 부분 가능 / ❌ 어려움

**근거**:
- (이유 설명)

**구현 방향** (가능한 경우):
- 어떤 도메인에 추가할 수 있는지
- 어떤 기술로 구현할 수 있는지
- 예상 API 엔드포인트

**검증 방법** (이력서 수치를 실제로 만들기 위한 테스트 방법):

단위/통합 테스트:
- 어떤 시나리오를 테스트해야 하는지 (동시성, 경쟁 조건 등)
- 사용할 테스트 도구 및 방법 (JUnit5, @SpringBootTest, CountDownLatch 등)
- 검증해야 할 핵심 조건 (중복 처리 여부, 데이터 정합성 등)

부하 테스트 (이력서 수치 확보):
- nGrinder 시나리오 구성 방법 (vUser 수, 요청 수, 측정 지표)
- Prometheus에서 확인할 메트릭 항목 (커넥션 사용률, 응답 시간 등)
- Before/After 비교를 위한 측정 기준점
```

### 3단계: 사용자 확인

아래 질문을 **한 번에** 묻는다:

```
## 확인 사항

1. 위 이력서 개선안 중 어떤 버전이 마음에 드시나요? (A/B/C 또는 직접 수정)
2. 이 기능을 프로젝트에 실제로 구현하시겠습니까?
   - 구현한다면 → 이슈 생성으로 연결합니다
   - 구현하지 않는다면 → 이력서 문장만 최종 정리해드립니다
```

---

### 4단계: 이슈 생성 (구현 결정 시)

사용자가 구현을 결정하면 `/issue` 워크플로와 동일하게 진행:

**불명확한 항목만** 추가로 질문:
- **대상**: 누가 사용하나? (USER / ADMIN / DORMITORY)
- **핵심 동작**: 정확히 어떤 일이 일어나야 하나?
- **범위**: 이번 이슈에 포함할 것 / 제외할 것

작업 목록 초안 제안 후 확인받고, GitHub MCP로 이슈 생성:
- owner: Teach-D
- repo: appcenter-16.5-dormitory-project
- title: `[{type}] {기능 설명}`
- body: 개요 + 작업 목록 + API 설계

브랜치 생성 및 로컬 체크아웃:
```bash
git fetch origin
git checkout -b teach/{type}/{설명}-{이슈번호} origin/teach/{type}/{설명}-{이슈번호}
```

`.claude/issues.md` 업데이트 후 커밋 & 푸시.

- 구현 완료 시 이력서에 추가할 수 있는 내용:
  [문제] {구현 후 더 강화된 문장}
  [해결] {구현 후 더 강화된 문장}
  [성과] {구현 후 더 강화된 문장}

```

구현하지 않는 경우:
```
## 결과 요약

### 이력서 문장 (최종)
