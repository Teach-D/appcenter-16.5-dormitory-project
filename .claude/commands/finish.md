현재까지 작업한 내용을 정리하고 커밋을 준비해줘.

다음 순서로 진행:

1. **변경 파일 확인**: `git diff --name-only` 와 `git status` 로 변경된 파일 목록 파악

2. **코드 설명**: 변경된 파일별로 무엇을 왜 바꿨는지 핵심만 설명
   - 새로 생성된 파일: 역할과 주요 로직 설명
   - 수정된 파일: 변경 전/후 차이와 이유 설명

3. **git add 승인 요청**: "위 파일들을 git add 할까요?" 라고 물어보고 대기
   - 승인하면 변경된 파일을 명시적으로 하나씩 `git add` (절대 `git add .` 사용 금지)

4. **커밋 메시지 3개 추천**: 아래 형식으로 번호 붙여서 제안
   ```
   1) feat: {영문 설명} #{이슈번호}
   2) feat: {다른 표현} #{이슈번호}
   3) {다른 prefix}: {또 다른 표현} #{이슈번호}
   ```
   - Conventional Commits 규칙 준수 (feat/fix/refactor/chore/docs 등)
   - 이슈 번호는 `.claude/issues.md`에서 현재 브랜치와 연결된 번호를 찾아서 사용
   - 이슈 번호를 특정할 수 없으면 번호 없이 제안

5. **커밋 실행**: 사용자가 번호를 선택하면 해당 메시지로 `git commit -m` 실행
