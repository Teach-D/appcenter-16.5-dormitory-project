현재 브랜치의 변경사항을 분석해서 PR 제목과 본문을 작성해줘. $ARGUMENTS

## 작업 순서

1. 현재 브랜치명과 base 브랜치(dev) 대비 변경사항 파악
```bash
git branch --show-current
git diff dev...HEAD --name-only
git log dev...HEAD --oneline
```

2. 변경된 파일들을 실제로 읽어서 내용 파악

3. 아래 형식으로 PR 초안 출력:

---

## PR 제목
`{type}: {한 줄 요약} #{이슈번호}`

타입 기준:
- `feat` - 새 기능
- `fix` - 버그 수정
- `refactor` - 리팩토링
- `chore` - 설정/빌드
- `docs` - 문서

## PR 본문

### 개요
(변경 목적을 2~3줄로 설명)

### 변경 사항
- (주요 변경 항목 bullet)
- ...

### 주요 로직
(핵심 구현 내용, 필요한 경우만)

### 테스트
- [ ] 로컬 빌드 확인
- [ ] (기능별 확인 항목)

### 관련 이슈
closes #(이슈번호)

---

## 주의사항
- 제목은 70자 이내
- 변경 파일이 많으면 도메인/레이어별로 그룹핑해서 설명
- 보안/인증 관련 변경이 있으면 반드시 언급
